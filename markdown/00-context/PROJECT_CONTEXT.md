# Project Context — Helima

## Project Name
Helima

## Product Type
Ecommerce website.

## Business Purpose
Helima sells products that are imported from China.

## Core Goal
Allow customers to browse products, sign in with Google, place orders, pay inside the website, and then continue communication through WhatsApp after payment.

## Required Stack
- Next.js for the web application
- Firebase for authentication, database, storage, and backend support
- Google Auth only
- Payment integration inside the app
- WhatsApp link after payment
- Admin dashboard

## Source of Truth
The Markdown files in this project folder are the source of truth.

The AI agent must not assume features that are not written here.

## MVP Principle
Build a clean, working, simple MVP first.
Do not overbuild.
Do not add marketplace, vendor accounts, delivery tracking, wallet, reviews, coupons, or AI features unless later added to the specs.
